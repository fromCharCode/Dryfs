//////////////
Conventions///
//////////////

Variables:
- begin with non-capital letters
- if possible the name should show what the var is for

Functions/Methods:
- begin with non-capital letters
- should specify the purpose

Classes:
- head of class should contain comments in following pattern with:
	- name of contributor
	- date of creation
	- if given a deadline
	- informations for other contributors
	- own notes 

	/*
	 * fromCharCode - David Schneider
	 * 03.06.2019
	 * -
	 * this class is .. on line X there is a bug, causing problem A, ...
	 *
	 * Notes: xyz, a + b = c, refactor methods foo(), bar(), ...
	*/
- classname begins with capital letter, each new word begins with capital as well e.g.: KonkreteKlasse.class


Packages:
- package names with non-capital letters
- you can use com.$name e.g.: com.fcc
